84 Custom validators 

85We will step through the evolution of writing a length-checking validator similar to the built-in Length validator, starting from a case-specific one to a generic reusable validator. 

86 

87Let’s start with a simple form with a name field and its validation: 

88 

89class MyForm(Form): 

90 name = TextField('Name', [Required()]) 

91 

92 def validate_name(form, field): 

93 if len(field.data) > 50: 

94 raise ValidationError('Name must be less than 50 characters') 

95Above, we show the use of an in-line validator to do validation of a single field. In-line validators are good for validating special cases, but are not easily reusable. If, in the example above, the name field were to be split into two fields for first name and surname, you would have to duplicate your work to check two lengths. 

96 

97So let’s start on the process of splitting the validator out for re-use: 

98 

99def my_length_check(form, field): 

100 if len(field.data) > 50: 

101 raise ValidationError('Field must be less than 50 characters') 

102 

103class MyForm(Form): 

104 name = TextField('Name', [Required(), my_length_check]) 

105All we’ve done here is move the exact same code out of the class and as a function. Since a validator can be any callable which accepts the two positional arguments form and field, this is perfectly fine, but the validator is very special-cased. 

106 

107Instead, we can turn our validator into a more powerful one by making it a factory which returns a callable: 

108 

109def length(min=-1, max=-1): 

110 message = 'Must be between %d and %d characters long.' % (min, max) 

111 

112 def _length(form, field): 

113 l = field.data and len(field.data) or 0 

114 if l < min or max != -1 and l > max: 

115 raise ValidationError(message) 

116 

117 return _length 

118 

119class MyForm(Form): 

120 name = TextField('Name', [Required(), length(max=50)]) 

121Now we have a configurable length-checking validator that handles both minimum and maximum lengths. When length(max=50) is passed in your validators list, it returns the enclosed _length function as a closure, which is used in the field’s validation chain. 

122 

123This is now an acceptable validator, but we recommend that for reusability, you use the pattern of allowing the error message to be customized via passing a message= parameter: 

124 

125class Length(object): 

126 def __init__(self, min=-1, max=-1, message=None): 

127 self.min = min 

128 self.max = max 

129 if not message: 

130 message = u'Field must be between %i and %i characters long.' % (min, max) 

131 self.message = message 

132 

133 def __call__(self, form, field): 

134 l = field.data and len(field.data) or 0 

135 if l < self.min or self.max != -1 and l > self.max: 

136 raise ValidationError(self.message) 

137 

138length = Length

143 WTForms 

144 WTForms – это мощная библиотека, написанная на Python и независимая от фреймворков. Она умеет генерировать формы, 

145 проверять их и предварительно заполнять информацией (удобно для редактирования) и многое другое.  

146 Также она предлагает защиту от CSRF. Для установки WTForms используется Flask-WTF. 

147 Flask- WTF – это расширение для Flask, которое интегрирует WTForms во Flask. Оно также предлагает дополнительные функции, 

148 такие как загрузка файлов, reCAPTCHA, интернационализация (i18n) и другие. Для установки Flask-WTF нужно ввести следующую команду pip install flask-wtf 

149  

150  

151 Пакет wtform предлагает несколько классов, представляющих собой следующие поля: 

152 StringField, PasswordField, SelectField, TextAreaField, SubmitField 

153  

154  

155 использовать несколько валидаторов, разделив их запятыми (,). Модуль wtforms.validators предлагает базовые валидаторы, 

156 но их можно создавать самостоятельно. В этой форме используются два встроенных валидатора: DataRequired и Email. 

157 DataRequired: он проверяет, ввел ли пользователь хоть какую-информацию в поле. 

158 Email: проверяет, является ли введенный электронный адрес действующим. 

159 Введенные данные не будут приняты до тех пор, пока валидатор не подтвердит соответствие данных.
